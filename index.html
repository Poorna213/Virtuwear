Frontend
Responsive interface built with plain HTML, CSS and JavaScript ‚Äì no frameworks. The demo page focuses on clarity and a minimal number of steps for the user.
Backend
Python Flask server exposes two main endpoints: /api/outfits to list garments and /api/tryon to generate the try-on image based on the selected outfit and uploaded photo.
Assets
Outfits are regular image files (e.g., .png / .jpg) inside assets/img_out. You can replace them with your own clothes.
Use Cases
Fashion experiments, mini e-commerce prototypes, styling previews or AI demo showcases.
How it works Flow
  • 1
    Load outfits: The demo page calls /api/outfits. The server scans assets/img_out and returns the list of available garments.
  • 2
    User input: You upload a photo and pick one outfit from the grid. This never leaves the app except for being sent to your own backend.
  • 3
    Request generation: The browser sends a multipart request to /api/tryon containing the photo, outfit filename and an optional style prompt.
  • 4
    AI processing: The backend loads both images, calls the configured AI image model and waits for the generated try-on result.
  • 5
    Preview: The server returns the final image. The browser displays it in the preview box, allowing you to view or save the result.
